Simple application installer

ОПИСАНИЕ

Программа предназначена для облегчения процесса установки приложений в Виндовс.
Пользователь выбирает необходимые приложения, после чего запускается процесс 
установки.

Что надо сделать, чтобы заставить все это работать

1. Загрузить дистрибутив программы
2. Выяснить, как запустить установку из командной строки
	Тут надо почитать документацию к программе. Очень полезным может 
	оказаться наличие возможности установки программы в так называемом 
	"тихом" (silent,quite,unattended) режиме, без вмешательства пользователя.
3. Добавить в файл настроек секцию [Application] и заполнить опции id, name, 
	command
4. Запустить Simple application installer
5. Выбрать нужное приложение
6. Нажать кнопочку Install

ФАЙЛЫ

	installer.js	запускает Simple application installer
	installer.cfg 	файл конфигурации
	conditions\*.js	скрипты для проверки условий
	
ТРЕБОВАНИЯ

КОНФИГУРАЦИЯ

Конфигурация хранится в файле installer.cfg, который должен присутствовать в 
папке со скриптом. Файл состоит из секций, внутри которых прописаны настройки.
Можно закомментировать строку, поставив в начале символ '#'. Установка 
приложений происходит в том порядке, в котором они записаны в файле.

секция [Options] содержит общие настройки скрипта
опции:
	default_template	идентификатор шаблона, используемого при 
				запуске скрипта
	window_width		ширина окна
	window_height		высота окна
	charset 		кодировка страницы (html)
	log			путь к логу
	row_height=7		количество приложений в одном столбце
	background_color	цвет фона
	wallpaper		wallpaper
	title			заголовок 

секция [Template] содержит шаблон, определяющий, какие приложения будут выбраны; 
таких секций может быть несколько
опции:
	id 		(обязателен) уникальный (еще раз: УНИКАЛЬНЫЙ) 
			идентификатор шаблона
	checked 	список идентификаторов приложений (через запятую, 
			без пробелов), которые должны быть выбраны 
секция [Application] описывает приложение; таких секций может быть несколько
опции:
	id 		(обязателен) уникальный (еще раз: УНИКАЛЬНЫЙ) 
			идентификатор приложения
	name		название приложения 
	command		команда для установки приложения (таких опций может 
			быть несколько)
	requires	список идентификаторов приложений (через запятую, без 
			пробелов), которые необходимы для работы данного 
			приложения
	conflicts	список идентификаторов приложений (через запятую, без 
			пробелов), которые не должны быть установлены с данным
	checkedif	условие (команда для запуска внешней программы), при 
			котором приложение должно быть выбрано независимо от 
			шаблона
	uncheckedif	условие (команда для запуска внешней программы), при 
			котором приложение не должно выбираться автоматически
	close_ie	закрывать internet explorer перед установкой приложения
			(значение данной опции значения не имеет, важно лишь 
			присутствие), полезно при установки некоторых программ,
			вроде флешплейера
	
ПЕРЕМЕННЫЙ ОКРУЖЕНИЯ

В опциях log,command,checkedif,uncheckedif могут использоваться переменные 
окружения операционной системы, например %temp%, %userprofile% и т.д. Кроме 
того, можно использовать вот такие переменные
	%root%			путь к папке со скриптом
	%defaultuserprofile%	путь к папке с профилем по-умолчанию
